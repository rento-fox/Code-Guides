Common Programming Naming Conventions and Their Typical Use Cases (by ChatGPT):

    Camel Case (camelCase)
        Definition: The first word starts with a lowercase letter,
        and each subsequent word starts with an uppercase letter. There are no spaces or underscores.
            Languages: JavaScript, Java, Swift, C#
            Purpose: variables, function names, method names, object properties.
            Example: userName, getUserInfo, userProfile

    Pascal Case (PascalCase)
        Definition: Similar to camel case, but the first word starts with an uppercase letter.
        No spaces or underscores.
            Languages: C#, Java, TypeScript, C++
            Purpose: class names, struct names, type names.
            Example: Person, UserProfile, OrderDetails

    Snake Case (snake_case)
        Definition: All letters are lowercase, and words are separated by underscores.
            Languages: Python, Ruby, PHP
            Purpose: variables, function names, method names, object properties.
            Example: user_name, get_user_info, user_profile

    Upper Snake Case (UPPER_SNAKE_CASE)
        Definition: Similar to snake_case, but all letters are uppercase.
            Languages: Python, C, C++, Java
            Purpose: constant variables, macro definitions.
            Example: MAX_VALUE, PI_CONSTANT, DEFAULT_TIMEOUT

    Kebab Case (kebab-case)
        Definition: All letters are lowercase, and words are separated by hyphens (-).
            Languages: JavaScript (for some libraries), CSS, URLs
            Purpose: CSS classes, file names, URLs.
            Example: user-profile, main-header, default-theme

    Hungarian Notation
        Definition: A prefix is added to the variable name to indicate its type or purpose
        (e.g., str for strings, int for integers).
            Languages: C, C++, older languages
            Purpose: Indicate the type of a variable.
            Example: strName, intCount, boolIsActive

    All Lowercase
        Definition: All letters are lowercase with no underscores or spaces.
            Languages: Shell scripts, URLs, Python
            Purpose: Often used for file names and sometimes for variable names in scripts or config files.
            Example: configfile, readme, setupscript

    Dot Notation (dot.notation)
        Definition: A naming convention where words are separated by periods (dots).
            Languages: Used in file paths, object properties in languages like JavaScript or Python.
            Purpose: Used to represent file paths or access object properties/methods.
            Example: com.example.project, user.profile.name

    Self-descriptive Naming
        Definition: Naming variables based on their purpose.
            Languages: All programming languages
            Purpose: Used when naming variables, functions, or classes to make their purpose clear.
            Example: totalAmount, calculateTax, getUserName, StudentClass
